# Hostile Prior Work

1. Contact-rich manipulation and task-and-motion planning already reason about constraints, but usually assume the relevant constraint set is observed, modeled, or manually specified.
2. Tactile and active-perception methods infer object properties, but often optimize information gain rather than a discrete hidden fixture mode that changes which release action is safe.
3. Assembly-planning work includes fixtures, jigs, clamps, and mating constraints, but these are typically design-time objects rather than latent causes discovered from failed motion.
4. Robot world models predict transitions, but a generic latent state is not the same as an interpretable fixture graph that names bolt, latch, adhesive, or hidden-stop causes.

The paper should therefore avoid claiming that hidden-state inference is new. The defensible novelty is the embodied variable: a latent fixture is a manipulable cause of constraint, inferred from outcome residuals and mapped to release actions.
