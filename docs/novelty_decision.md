# Novelty Decision

Decision: proceed as a mechanism paper, not as a benchmark paper.

The strongest version is not larger perception, more data, or a new planner. It is a different state variable for manipulation: latent fixture cause. A robot that sees a part fail to move should ask whether it is bolted, latched, adhered, blocked by a hidden stop, or actually free. That hidden cause changes the next safe action. The toy experiment is intentionally small, but it demonstrates the mechanism cleanly.

V2 boundary: proceed only as workshop-only / strong-revise. The out-of-taxonomy cable-tie stress shows that the mechanism depends on a calibrated fixture taxonomy and release-action menu; when those are incomplete, the safe policy must abstain rather than claim task success.
